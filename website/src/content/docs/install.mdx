---
title: Build Nagi from source
description: Build and verify Nagi while signed public binaries are not available.
---

Nagi does not publish signed binaries or package-manager releases yet. The
inherited download channels are disabled so they cannot install an upstream or
unsigned binary under the Nagi name.

## Requirements

- Rust 1.96.1
- Zig 0.15.2
- a Unix system for the mission runtime

## Build

```bash
git clone https://github.com/Cod-Hash-Studios/nagi.git
cd nagi

zig version  # must print 0.15.2
cargo build --release --locked
./target/release/nagi
```

## Verify

```bash
./target/release/nagi --version
./target/release/nagi api schema
```

Review [the current status](https://github.com/Cod-Hash-Studios/nagi#the-honest-status)
before relying on Nagi for important work.

## Updates

Pull a reviewed commit, rebuild with the pinned toolchain, and restart Nagi.
Automatic updates remain disabled until the project publishes signed artifacts
from its own release pipeline.
