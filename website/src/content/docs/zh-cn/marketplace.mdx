公开的 Nagi 插件注册表仍在规划中,尚未上线。
[注册表页面](/plugins/)只展示当前状态,不会把继承自上游或未经验证的仓库
列为 Nagi 插件。

## 现在可用的功能

本地插件开发和从你已信任的 GitHub 仓库直接安装都可以使用:

```bash
nagi plugin install owner/repo[/subdir...]
```

仓库根目录或指定子目录必须包含 `nagi-plugin.toml`。开发时也可以链接本地
checkout。命令、清单格式和[信任与安全](/zh-cn/docs/plugins/#信任与安全)
请见[插件](/zh-cn/docs/plugins/)。

## 尚不可用的功能

目前没有 `nagi-plugin` 主题的自动索引、定时刷新或公开提交流程。现在给仓库
添加主题标签,不会让它出现在 Nagi 网站上。

上线前,注册表需要明确的信任模型、来源展示、滥用处理机制和清晰的审核边界。
进展会发布在 [Nagi 仓库](https://github.com/Cod-Hash-Studios/nagi)。
