The public Nagi plugin registry is planned, but it is not live yet. The
[registry page](/plugins/) tracks that status without listing inherited or
unverified repositories as Nagi plugins.

## What works today

Local plugin development and direct GitHub installation work. Install a plugin
from a repository you already trust:

```bash
nagi plugin install owner/repo[/subdir...]
```

The repository needs a `nagi-plugin.toml` manifest at its root or in the chosen
subdirectory. You can also link a local checkout while developing. See
[Plugins](/docs/plugins/) for the commands, manifest format, and
[trust guidance](/docs/plugins/#trust-and-security).

## What does not work yet

There is no automatic `nagi-plugin` topic index, listing refresh schedule, or
public submission flow. Adding a topic to a repository will not make it appear
on the Nagi site today.

Before launch, the registry needs an explicit trust model, provenance display,
abuse handling, and clear review boundaries. Progress will be published on the
[Nagi repository](https://github.com/Cod-Hash-Studios/nagi).
