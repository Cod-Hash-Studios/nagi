公開 Nagi プラグインレジストリは計画中で、まだ稼働していません。
[レジストリページ](/plugins/)では、継承元や未検証のリポジトリを Nagi
プラグインとして掲載せず、現在の状況を案内しています。

## 現在使える機能

ローカルでのプラグイン開発と、信頼済み GitHub リポジトリからの直接
インストールは利用できます:

```bash
nagi plugin install owner/repo[/subdir...]
```

リポジトリのルートまたは指定したサブディレクトリに `nagi-plugin.toml` が必要です。
開発中はローカルチェックアウトをリンクすることもできます。コマンド、
マニフェスト形式、[信頼とセキュリティ](/ja/docs/plugins/#信頼とセキュリティ)は
[プラグイン](/ja/docs/plugins/)を参照してください。

## まだ使えない機能

`nagi-plugin` トピックの自動インデックス、定期更新、公開申請フローはありません。
現在、リポジトリにトピックを追加しても Nagi サイトには掲載されません。

公開前に、レジストリには明確な信頼モデル、出所表示、不正利用への対応、
レビュー境界が必要です。進捗は
[Nagi リポジトリ](https://github.com/Cod-Hash-Studios/nagi)で公開します。
